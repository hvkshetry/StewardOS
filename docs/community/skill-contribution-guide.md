# Skill Contribution Guide

This guide defines how to contribute high-value persona skill improvements to StewardOS.

## Purpose

StewardOS is most useful when domain experts contribute practical, auditable workflows.

Examples:

- nutrition professionals improving Wellness Advisor skill quality,
- professional investors improving Investment Officer decision scaffolds,
- CPAs/bookkeepers improving Household Comptroller checks and controls,
- legal/estate professionals improving Estate Counsel workflows.

## Contribution requirements

Each skill contribution should include:

1. Problem statement and intended user outcome.
2. Updated skill instructions with clear triggers.
3. Tool/provenance expectations.
4. Concrete input/output examples.
5. Risk boundaries and escalation behavior.

## Quality checklist

- role-appropriate for persona scope,
- no private identifiers or live secrets,
- deterministic enough for automated usage,
- explicit data quality expectations,
- backwards-compatible with current persona contracts unless migration notes are provided.

## Recommended PR structure

1. Skill changes.
2. Before/after examples.
3. Validation notes and assumptions.
4. Any required doc updates in architecture or contributing docs.
